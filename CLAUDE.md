# CLAUDE.md — Xouston Landing Page

## О проекте

Лендинг для IT-студии **Xouston** (xouston.com). Мы занимаемся разработкой мобильных и веб-приложений под ключ, предоставляем DevOps-услуги, настраиваем инфраструктуру и отказоустойчивость. Также предоставляем аутсорс-услуги — выделяем разработчиков под проект заказчика (аутстаффинг). Позиционирование: надёжная продуктовая студия с глубокой экспертизой, которая одинаково хорошо ведёт проекты под ключ и усиливает существующие команды.

## Стек проекта

- **Framework:** Next.js 14 (App Router)
- **Language:** TypeScript
- **Styling:** Tailwind CSS
- **i18n:** next-intl (RU / EN, русский — основной)
- **Анимации:** Framer Motion
- **Деплой:** Vercel
- **Линтинг:** ESLint + Prettier

## Дизайн-система

### Референс

Минималистичный, светлый, карточный дизайн (см. скриншот референса в `/docs/reference.webp`). Ключевые черты:
- Светлый фон (#FAFAFA / white), тёмный текст
- Карточки с мягкими тенями и скруглениями (border-radius: 16px)
- Сетка карточек в hero-секции (bento-grid стиль)
- Крупная типографика для заголовков
- Акцентный цвет — синий (#2563EB) для CTA-кнопок и ссылок
- Много воздуха между секциями
- Плавные hover-анимации на карточках

### Шрифты

- **Заголовки:** Inter (700, 600)
- **Текст:** Inter (400, 500)
- Подключать через `next/font/google`

### Цветовая палитра (Tailwind config)

```
colors: {
```

### Компоненты

- Все компоненты — функциональные React-компоненты с TypeScript
- Каждый компонент — отдельный файл в `/src/components/`
- Секции лендинга — в `/src/sections/`
- Переиспользуемые UI-элементы (Button, Card, Badge, SectionHeading) — в `/src/components/ui/`

## Структура страниц

### 1. Главная (Landing Page)

**Навигация (Header):**
- Логотип "XO Xouston" слева
- Меню: Услуги, Подход, Кейсы, Блог, Контакты
- Переключатель RU / EN справа
- Фиксированный header с blur-фоном при скролле

**Hero-секция (Bento Grid):**
- Главная карточка: заголовок + подзаголовок + CTA-кнопки ("Оставить заявку", "Быстрая оценка")
- Бейджи: "NDA-first подход", "Полный цикл", "Готовность к EU"
- Подпись: "Ответ в течение 1 рабочего дня. Работаем по NDA."
- Дополнительные карточки в bento-сетке:
  - 24/7 поддержка
  - RU / EN — фокус рынков
  - Рабочий ритм: прозрачные статусы, еженедельные демо, фокус на KPI
  - Визуальная карточка с абстрактной графикой или паттерном

**Метрики / Кейсы под NDA:**
- Заголовок: "Анонимные результаты по проектам под NDA"
- 3 карточки с крупными цифрами:
  - Кейс 01: метрика конверсии / роста
  - Кейс 02: метрика оптимизации / снижения затрат
  - Кейс 03: метрика ускорения разработки
- Цифры — крупный шрифт, подписи мелким текстом

**Услуги:**
- Карточки с нумерацией (01, 02, 03...)
- Мобильная разработка (iOS, Android, кроссплатформа: React Native, Flutter, KMP, Swift, Kotlin)
- Сайты и веб-платформы (React, Vue, Next.js, панели управления, SEO)
- DevOps и инфраструктура (CI/CD, мониторинг, отказоустойчивость, масштабирование)
- Telegram и чат-боты (автоматизация, интеграции)
- Backend-разработка (Python/Django/FastAPI, PHP/Symfony/Laravel, Go, микросервисы)
- Аудит и консалтинг (код-ревью, архитектурный аудит, оптимизация)
- Аутстаффинг (выделенный разработчик или команда под проект заказчика, интеграция в существующие процессы)

**Подход / Как мы работаем:**
- Шаги: Брифинг → Оценка → Дизайн → Разработка → Тестирование → Запуск → Поддержка
- Визуально: вертикальный таймлайн или горизонтальные карточки

**Технологии:**
- Логотипы стека с подписями, сгруппированные по категориям (Mobile / Frontend / Backend / DevOps)

**Почему мы:**
- 3–4 карточки с преимуществами:
  - NDA-first: конфиденциальность по умолчанию
  - Полный цикл: от идеи до поддержки
  - Прозрачность: еженедельные демо и отчёты
  - Гибкость: проект под ключ или усиление вашей команды

**CTA-секция:**
- "Обсудим ваш проект?" + кнопка + email/Telegram

**Footer:**
- Логотип, навигация, контакты, соцсети, копирайт

### 2. Кейсы (/cases)

- Карточки проектов с фильтрацией по категориям (mobile, web, devops)
- Каждый кейс: обложка, название, краткое описание, теги технологий, ключевая метрика
- Страница кейса: задача → решение → результат → стек
- Для NDA-проектов: анонимные карточки с метриками без имён клиентов

### 3. Блог / Новости (/blog)

- Список статей с превью
- Категории: разработка, DevOps, бизнес, кейсы
- SEO-оптимизированные статьи (meta title, description, OG)
- Markdown-контент (MDX или CMS позже)

### 4. FAQ (/faq или секция на главной)

- Аккордеон с вопросами:
  - Сколько стоит разработка?
  - Как долго длится разработка?
  - Работаете ли вы по NDA?
  - Какие гарантии вы даёте?
  - Можно ли подключить вашу команду к существующему проекту?
  - Как устроена поддержка после запуска?

### 5. Контакты (/contacts)

- Форма обратной связи (имя, email/телефон, описание проекта)
- Email, Telegram, WhatsApp
- Опционально: Calendly-виджет для букинга звонка

### 6. О компании (/about) — опционально

- Миссия и ценности
- Команда (можно без фото — иконки/аватары)
- Цифры: размер команды, количество технологий в стеке, география клиентов

## Правила по SEO

- Каждая страница: уникальный `<title>`, `<meta description>`, OG-теги
- Семантическая вёрстка: h1 один на страницу, h2–h3 по иерархии
- alt-тексты на всех изображениях
- JSON-LD разметка (Organization, FAQ, BreadcrumbList)
- sitemap.xml и robots.txt
- Чистые URL: `/cases/mobile-app-b2c`, не `/cases/1`
- Скорость: оптимизация изображений (next/image), lazy loading

## Тон и копирайтинг

- Профессиональный, уверенный, без воды
- Не хвастаемся количеством проектов — делаем акцент на экспертизе руководства и качестве подхода
- Не упоминаем возраст студии и точное количество завершённых проектов
- Формулировки вроде: "сильная команда", "проверенные процессы", "глубокая экспертиза в стеке"
- НЕ писать про "многолетний опыт руководства" — звучит топорно. Экспертиза должна считываться через подход, кейсы и стек, а не через прямые заявления
- Метрики в кейсах — анонимные, через проценты и множители (+34%, 2.3x)
- CTA — мягкие, не агрессивные: "Обсудим ваш проект", "Получить оценку"

## Важные ограничения

- НЕ указывать, что студии 1 год
- НЕ писать точное число завершённых проектов (2)
- НЕ использовать стоковые фото людей — лучше абстрактная графика, паттерны, иконки
- НЕ обещать то, чего не можем (99.99% uptime и т.п.) — писать мягче
- Весь текст должен быть и на RU, и на EN — хранить в messages/*.json
- Мобильная адаптивность — обязательно, mobile-first
- Тёмная тема — НЕ нужна (только светлая)
